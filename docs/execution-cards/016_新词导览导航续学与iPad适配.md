# 执行卡 016｜新词导览导航、续学与 iPad 适配

状态：实施中
创建日期：2026-07-24
目标分支：`agent/vocabulary-lesson-navigation`
关联功能：新词导览 / 常用词单词本

## 一、背景

新词导览上线后，实际课堂使用发现以下问题：

1. 单词本列表中的绿色背景含义不明确。
2. 单词本顺序与预期不一致，新日期单词本会浮到列表顶部。
3. iPad 横屏查看真实图片时，图片下半部分可能被截断。
4. 页面没有批内进度提示。
5. 只能从第一批顺序进入，无法直接选择第二、第三、第四批。
6. 难词巩固和随机抽取只能在全部批次结束后进入。
7. 退出后重新打开会重新回到第一批，不能接续上次课堂位置。

## 二、产品目标

在不重做现有新词导览主体结构的前提下，完成以下优化：

- 单词本按时间从旧到新展示。
- 最新单词本保持原时间位置，但使用薄荷绿色背景和“最新”标签强调。
- 顶部增加紧凑批次导航，可随时切换批次、难词巩固和随机抽取。
- 每批显示圆点式进度，不显示整个单词本总词数或剩余词数。
- 保存每个用户、每个单词本的课堂进度，重新进入时自动续学。
- iPad 横屏中完整显示图片，允许留白但不允许裁掉图片内容。

## 三、单词本列表规则

### 3.1 排序

单词本严格按有效日期升序排列：

1. `sortDate`
2. `date`
3. `createdAt`
4. `updatedAt`

相同日期时按 `id` 升序作为稳定排序。

### 3.2 最新单词本强调

- 最新单词本仍位于列表最后，不改变排序位置。
- 使用薄荷绿色背景。
- 显示明确的“最新”小标签。
- 不再使用“列表第一项即绿色”的隐式规则。
- 绿色只表达“这是最新单词本”，不表达完成状态或推荐状态。

## 四、顶部导航

顶部导航采用：

`① ② ③ ④  ★ 难词  ↻ 随机`

### 4.1 批次按钮

- `①` 至 `④` 分别代表第一至第四批。
- 当前批次使用实心薄荷绿色状态。
- 其他可进入批次使用白底描边。
- 当前单词本不存在的批次显示为禁用状态。
- 点击批次按钮立即进入该批，不要求先完成前一批。
- 每批默认恢复该批上次看到的词。

### 4.2 难词巩固

- 顶部始终保留入口。
- 没有难词时禁用。
- 有难词时可随时进入。
- 难词标记不得因退出页面、刷新页面或切换批次而丢失。

### 4.3 随机抽取

- 顶部始终保留入口。
- 点击后从当前单词本生成随机词组。
- 进入随机模式后顶部仍保留所有导航入口。

## 五、批内圆点进度

- 一个圆点对应当前批次中的一个词。
- 已看过：浅薄荷绿色实心圆点。
- 当前词：粉色放大圆点。
- 未看：空心圆点。
- 不显示以下信息：
  - 整个单词本共有多少词；
  - 总共还剩多少词；
  - `3 / 40` 等跨批次数字。
- 可为辅助功能提供“本批第几个词”的 `aria-label`，但不在视觉界面显示数字。

## 六、续学状态

### 6.1 保存范围

按“用户 + 单词本”分别保存：

- 上次所在批次；
- 每一批上次所在词序号；
- 难词列表。

### 6.2 恢复规则

- 重新进入单词本时，恢复上次所在批次和词。
- 点击其他批次后，恢复该批上次所在词。
- 首次进入没有记录时，从第一批第一个词开始。
- 已删除或数量发生变化的词不应导致越界或空白页面。

### 6.3 存储

- 课堂进度使用 `localStorage`。
- 难词状态由原 `sessionStorage` 迁移至 `localStorage`。
- 发现旧会话难词记录时自动迁移一次。

## 七、iPad 图片适配

### 7.1 目标

在 iPad 横屏等长方形视口中：

- 图片完整显示；
- 不裁掉图片上部或下部；
- 允许图片周围出现留白；
- 底部操作按钮仍完整可见；
- 页面不得产生横向滚动。

### 7.2 实现原则

- 场景图使用 `object-fit: contain`。
- 图片同时受 `max-width: 100%` 和 `max-height: 100%` 限制。
- 图片定位在视觉容器内部，不由自身固有尺寸撑开布局。
- iPad 横屏调整图片区与信息区比例，优先保证视觉区完整。
- 不使用 `cover`，不通过裁切解决比例问题。

## 八、实施方式

采用独立 UX 补丁，避免大范围重写已经稳定的新词导览：

- `js/vocabularyLessonUxPatch.js`
  - 排序和最新单词本标识；
  - 顶部批次导航；
  - 圆点进度；
  - 续学状态；
  - 难词持久化。
- `styles-vocabulary-lesson-patch.css`
  - 最新单词本样式；
  - 批次按钮和进度圆点；
  - iPad 图片完整显示规则。
- `js/vocabularyReviewData.js`
  - 自动加载 UX 补丁文件。
- `data/vocabularyLessonAssets.js`
  - 将新增补丁加入离线缓存资源。
- `service-worker.js`
  - 更新缓存版本。
- 新增对应单元测试和 WebKit 视口测试。

## 九、验收标准

### 9.1 单词本列表

- [ ] 所有单词本从旧到新排列。
- [ ] 最新单词本位于正常时间位置，不被强行移到顶部。
- [ ] 只有最新单词本显示绿色背景和“最新”标签。

### 9.2 顶部导航

- [ ] 顶部显示 `① ② ③ ④ ★ 难词 ↻ 随机`。
- [ ] 不存在的批次不可点击。
- [ ] 当前模式或批次有明确选中状态。
- [ ] 任意时刻均可切换至已有批次。

### 9.3 进度

- [ ] 当前批次显示圆点进度。
- [ ] 不显示总词数、剩余总数或跨批次数字进度。
- [ ] 切换词时圆点即时更新。

### 9.4 续学

- [ ] 退出后重新进入恢复上次批次。
- [ ] 恢复该批上次词位置。
- [ ] 每个单词本独立保存进度。
- [ ] 每个用户独立保存进度。
- [ ] 难词刷新后仍保留。

### 9.5 iPad

- [ ] 1180 × 820 WebKit 视口中真实图片四边完整可见。
- [ ] iPad 横屏底部按钮完整可见。
- [ ] 页面没有横向溢出。
- [ ] 图片回顾墙仍可正常显示和进入详情。

## 十、不在本卡范围

- 不改变单词卡数据结构。
- 不改变每批默认 10 词的规则。
- 不重新设计新词导览完整页面。
- 不改变图片生成和导入流程。
- 不增加全单词本总进度或剩余数量。
