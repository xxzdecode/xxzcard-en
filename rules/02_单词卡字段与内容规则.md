# 02_单词卡字段与内容规则

更新时间：2026-07-10  
状态：正式规则  
适用范围：英语单词卡内容生成、旧数据审核、内容迁移与质量检查

> 本文件只规定单词卡的字段结构、字段含义和内容质量。  
> 本文件不是最终导入文本格式。实际导入时的换行、空行、数组序列化和解析要求，以《03_单词卡真实导入格式》为准。

---

## 1. 使用对象与总体原则

本规则适用于：

- 小学生
- 英语基础薄弱的初中生
- 英语单词卡内容生成
- 旧版单词卡数据迁移
- 单词卡内容审核与修正
- 后续构词、音标和专项练习的数据支持

总体原则：

1. 所有字段名称固定，不随意新增、删除或改名。
2. 所有音标统一使用英式 IPA。
3. 内容应简短、自然、准确，适合学生理解。
4. 只填写真正有教学价值的内容。
5. 不为了填满字段而硬编搭配、词族、近义词或构词知识。
6. 同一知识点不要在多个字段中重复解释。
7. 字段内容必须与当前词义、词性和音标保持一致。
8. 增强版强调内容质量和实用性，不等于每个字段都必须有内容。

---

## 2. 固定字段与字段顺序

每张单词卡固定使用以下字段：

```json
{
  "word": "",
  "meaning": "",
  "pos": "",
  "phonetic": "",
  "emoji": "",
  "morphology": [],
  "collocations": [],
  "irregularForms": [],
  "synonyms": [],
  "wordFamily": [],
  "tip": ""
}
```

字段顺序固定为：

1. `word`
2. `meaning`
3. `pos`
4. `phonetic`
5. `emoji`
6. `morphology`
7. `collocations`
8. `irregularForms`
9. `synonyms`
10. `wordFamily`
11. `tip`

禁止使用旧字段：

- `en`
- `zh`
- `ex`
- `note`

---

## 3. 基础字段规则

### 3.1 word

`word` 表示单词本体。

要求：

- 动词默认使用原形，如 `go`、`eat`、`play`。
- 可数名词默认使用单数，如 `child`、`box`、`apple`。
- 形容词默认使用原级，如 `happy`、`big`。
- 副词使用标准词形，如 `quickly`、`well`。
- 不把普通复数、三单或过去式自动作为主词条。
- 只有当某个变形已经形成独立高频词义或有明确教学需要时，才可单独作为词条。
- 拼写必须先检查，不能直接保留明显录入错误。

示例：

```json
"word": "wind"
```
#### 3.1.1 多词输入与核心词选择

原始输入可能是单个单词、短语动词、固定表达或普通搭配。原始输入不一定直接等于 `word` 字段。

默认规则：

1. `word` 优先保存一个核心单词。
2. 输入为多词表达时，必须先确定核心词，再生成单词卡。
3. 原始多词表达应保留在核心词卡片的 `collocations` 中。
4. `meaning`、`pos` 和 `phonetic` 必须对应核心词本身，不能填写整个短语的词义、词性或音标。
5. 同一个核心词在同一批次中只能生成一张卡片。

短语动词以主要动词作为核心词：

```text
figure out → figure
find out → find
give up → give
look after → look
```

含介词、连词、副词等功能词的固定表达，应选择承担主要词义的实词作为核心词：

```text
as well → well
at first → first
in person → person
```

普通的形容词加名词、名词加名词或动词加宾语结构，默认拆成需要学习的独立单词：

```text
average temperature → average + temperature
wind direction → wind + direction
solve problem → solve + problem
clever idea → clever + idea
```

完整表达可以加入相关核心词的 `collocations`，但不得直接作为普通 `word` 主词条。

如果输入同时包含核心词和相关表达，例如：

```text
find
find out
```

最终只生成一张 `find` 卡片，并把 `find out` 合并到 `find` 的 `collocations` 中。

多个表达映射到同一个核心词时，应合并到同一张卡片，不得重复生成。

只有符合以下情况时，`word` 才可以保留多词形式：

* 该表达本身是不可自然拆分的固定词项；
* 拆分后会失去主要词义；
* 用户明确要求把完整表达作为主词条收录。

不能仅仅因为原始输入中含有空格，就直接把整个表达写入 `word`。

---

### 3.2 meaning

`meaning` 表示当前词卡需要教学的核心中文义。

要求：

- 优先写最常用、最适合当前阶段的意思。
- 中文应简短、自然、便于学生理解。
- 不堆积大量词典释义。
- 多个紧密相关的常用义可以用中文分号分隔。
- 同形异音、意义差异很大的用法，不应随意塞进同一张卡。
- `meaning` 必须与 `pos`、`phonetic` 和其他字段保持一致。

示例：

```json
"meaning": "风"
```

多义示例：

```json
"meaning": "跑；经营"
```

只有在当前卡片确实同时教授两个意思时，才使用多义写法。

---

### 3.3 pos

`pos` 表示词性。

统一使用以下缩写：

| 词性 | 写法 |
|---|---|
| 名词 | `n.` |
| 动词 | `v.` |
| 形容词 | `adj.` |
| 副词 | `adv.` |
| 介词 | `prep.` |
| 连词 | `conj.` |
| 代词 | `pron.` |
| 限定词 / 冠词 | `det.` |
| 感叹词 | `interj.` |

可以使用组合词性，例如：

```json
"pos": "n./v."
```

但只有当卡片中的中文义、搭配和例句确实同时覆盖这些词性时，才允许组合。不能只因为词典中还存在其他词性，就全部列出。

---

### 3.4 phonetic

`phonetic` 表示英式国际音标。

要求：

- 只使用英式 IPA。
- 音标必须放在 `/ /` 中。
- 不同时写英式和美式音标。
- 使用 `/əʊ/`，不使用 `/oʊ/`。
- 使用英式 `/ɒ/` 等标记。
- 不使用美式卷舌或简化写法替代英式发音。
- 当前词义对应不同读音时，只写当前词义所对应的读音。
- 音标不确定时必须核实，不能猜测。
- 单词卡音标应与音标训练分类保持一致。

示例：

```json
"phonetic": "/wɪnd/"
```

如 `wind`：

- “风”读 `/wɪnd/`
- “缠绕；上发条”读 `/waɪnd/`

不同读音和词义不应混在同一张基础卡片中。

---

### 3.5 emoji

`emoji` 表示帮助学生理解词义的直观图标。

要求：

- 一般使用一个 emoji。
- 优先选择能直接表达核心词义的图标。
- 不使用容易误导的图标。
- 不为抽象词强行选择不相关 emoji。
- 没有合适图标时使用空字符串。

示例：

```json
"emoji": "📖"
```

无合适图标时：

```json
"emoji": ""
```

---

## 4. morphology：构词知识

### 4.1 字段结构

```json
"morphology": [
  {
    "part": "",
    "meaning": "",
    "knowledgeKey": ""
  }
]
```

每一条只表示一个明确的构词知识点。

示例：

```json
"morphology": [
  {
    "part": "un-",
    "meaning": "表示不、相反：happy → unhappy",
    "knowledgeKey": "prefix-negative"
  }
]
```

---

### 4.2 可以填写的内容

可以记录：

- 简单、常见的前缀
- 简单、常见的后缀
- 非常清楚且适合学生理解的简单词根
- 明显且有教学价值的合成词结构
- 能帮助学生记忆的真实拼写变化

前提：

1. 当前单词本身真实包含该结构。
2. 对小学生或初中基础薄弱学生有帮助。
3. 解释短而清楚。
4. 后续可以用于构词专项分类。

---

### 4.3 禁止事项

禁止：

- 牵强拆词。
- 把字母巧合当作词根或词缀。
- 做复杂词源分析。
- 把派生词中才存在的词缀写进当前词。
- 把多个不同构词点挤在同一个对象里。
- 与 `tip` 重复解释同一内容。
- 临时创造未统一的 `knowledgeKey`。

例如，`wind` 本身不包含 `-y`，因此不能因为存在 `windy` 就在 `wind` 的 `morphology` 中写 `-y`。

没有适合内容时：

```json
"morphology": []
```

---

### 4.4 多个构词点

一个词中有多个真实且有教学价值的构词点时，应拆成多条。

示例：

```json
"morphology": [
  {
    "part": "un-",
    "meaning": "表示不、相反：fortunate → unfortunate",
    "knowledgeKey": "prefix-negative"
  },
  {
    "part": "-ly",
    "meaning": "加在形容词后，常变成副词：unfortunate → unfortunately",
    "knowledgeKey": "suffix-adverb"
  }
]
```

---

### 4.5 knowledgeKey 建议

常用分类：

| 类型 | knowledgeKey |
|---|---|
| 否定 / 相反前缀 | `prefix-negative` |
| 再次 / 重复前缀 | `prefix-again` |
| 形容词后缀 | `suffix-adjective` |
| 名词后缀 | `suffix-noun` |
| 人 / 职业后缀 | `suffix-person` |
| 副词后缀 | `suffix-adverb` |
| 比较级 / 最高级 | `suffix-comparison` |
| 拼写变化 | `spelling-change` |
| 合成词 | `compound-word` |

需要新增分类时，应统一更新规则，不要单次任务中随意命名。

## 5. collocations：常用搭配与例句

### 5.1 字段结构

```json
"collocations": [
  {
    "phrase": "",
    "example": ""
  }
]
```

---

### 5.2 内容范围

`collocations` 可以包括：

- 固定表达
- 高频自然搭配
- 常用短语
- 常用句型片段
- 对学生口语或写作有复用价值的表达

不要求每一条都必须是严格语言学意义上的“固定搭配”，但必须自然、常见、有教学价值。

---

### 5.3 数量

每个单词提供 **1–5 个**搭配。

具体数量根据以下因素决定：

- 单词是否高频
- 是否有多个常见用法
- 是否为常用动词
- 是否有多个值得掌握的句型
- 学生当前学习阶段
- 卡片整体信息量

规则：

- 不为凑数量写低频、生硬或重复搭配。
- 多个搭配不能只是替换一个普通名词、实际表达同一用法。
- 高频、多义词可以接近 5 个。
- 用法单一的普通词可以只写 1 个。
- 增强版强调实用和完整，不以固定数量机械判断质量。

---

### 5.4 搭配选择原则

优先选择：

1. 高频、自然的表达。
2. 学生在日常口语或写作中能复用的表达。
3. 能体现当前词真实用法的表达。
4. 能帮助区分词义或词性的表达。
5. 与当前教学内容相关的表达。

禁止：

- 随机把两个词放在一起。
- 用普通句子冒充搭配。
- 写太成人化、太抽象或太低频的表达。
- 多条搭配重复同一知识点。
- 为了显得丰富而加入学生暂时不需要的复杂表达。

---

### 5.5 例句要求

每个搭配必须有一个对应例句。

格式：

```json
{
  "phrase": "read a book",
  "example": "I read a book every day. / 我每天读一本书。"
}
```

要求：

- 例句必须真正使用对应搭配。
- 英文和中文意思一致。
- 句子自然，不使用生硬直译。
- 难度适合小学生或初中基础薄弱学生。
- 一般控制在约 5–12 个英文单词。
- 尽量使用学生已经学过的词和句型。
- 中英文写在同一个 `example` 中。
- 英文和中文之间使用 ` / ` 分隔。
- 不额外增加普通例句字段。

---

## 6. irregularForms：不规则形式

### 6.1 字段结构

固定使用：

```json
"irregularForms": [
  {
    "label": "",
    "form": ""
  }
]
```

示例：

```json
"irregularForms": [
  {
    "label": "过去式",
    "form": "go → went"
  },
  {
    "label": "过去分词",
    "form": "go → gone"
  }
]
```

---

### 6.2 允许的 label

优先使用：

- `现在式`
- `过去式`
- `过去分词`
- `三单`
- `复数`
- `比较级`
- `最高级`

---

### 6.3 只写真正不规则的形式

应该写：

```json
{"label":"过去式","form":"go → went"}
```

```json
{"label":"三单","form":"have → has"}
```

```json
{"label":"复数","form":"child → children"}
```

```json
{"label":"比较级","form":"good → better"}
```

不要写：

- `play → played`
- `book → books`
- `wind → winds`
- `small → smaller`
- `small → smallest`

普通规则变化不放进 `irregularForms`。

没有不规则形式时：

```json
"irregularForms": []
```

---

## 7. synonyms：近义词

### 7.1 字段结构

```json
"synonyms": [
  {
    "word": "",
    "meaning": ""
  }
]
```

---

### 7.2 收录原则

只收录：

- 真正的近义词
- 意义高度接近、值得学生对比的词

不收录：

- 普通主题相关词
- 同一场景中的其他词
- 反义词
- 仅仅拼写相似的词
- 含义相距较远的“相关词”

数量：

- 通常 0–2 个。
- 少数确有教学价值时，可以增加。
- 最多不超过 5 个。
- 没有合适内容时使用空数组。

示例：

```json
"synonyms": [
  {
    "word": "sad",
    "meaning": "难过的"
  }
]
```

字段名继续使用 `synonyms`，前端不应把普通相关词混入这一字段。

---

## 8. wordFamily：词族

### 8.1 字段结构

```json
"wordFamily": [
  {
    "word": "",
    "pos": "",
    "meaning": ""
  }
]
```

示例：

```json
"wordFamily": [
  {
    "word": "teacher",
    "pos": "n.",
    "meaning": "老师"
  }
]
```

---

### 8.2 收录原则

只收录：

- 与主词具有真实派生关系的词。
- 常用、清楚、适合当前学生的词。
- 能帮助理解词性变化或构词规律的词。

优先示例：

- `happy → happiness`
- `teach → teacher`
- `sing → singer`
- `help → helpful`
- `care → careful → carefully`

禁止：

- 普通主题相关词。
- 仅拼写相似、意义无关的词。
- 太难、太学术或太低频的派生词。
- 为了填字段而机械加入最基础的反向关系。
- 将 `unhappy → happy` 作为必须补充的词族内容。

数量：

- 通常 0–3 个。
- 没有合适内容时使用空数组。

---

## 9. tip：易混点与简短提醒

### 9.1 字段类型

`tip` 固定为字符串：

```json
"tip": ""
```

---

### 9.2 应写内容

出现以下情况时应填写：

- 常见易混词。
- 发音容易出错。
- 拼写容易看混。
- 用法存在高频误区。
- 中英文表达方式不同。
- 同形词、近形词或近音词容易混淆。
- 当前词有学生经常犯的错误。

示例：

```json
"tip": "sleepy 表示“想睡的”，asleep 表示“已经睡着的”。"
```

---

### 9.3 禁止内容

禁止：

- 重复 `meaning`。
- 重复 `morphology` 已经解释的词缀。
- 重复 `irregularForms`。
- 重复搭配和例句。
- 写“这是一个常用词”等无信息量内容。
- 写与当前词无关的文化知识。
- 写长篇语法解释。

没有真正值得提醒的内容时：

```json
"tip": ""
```

---

## 10. 字段之间的去重原则

同一知识点原则上只解释一次。

### morphology 与 tip

如果 `morphology` 已经说明：

```text
un- 表示“不、相反”
```

`tip` 不应再次重复这一点。

### wordFamily 与 morphology

- `morphology` 解释当前单词本身的构词结构。
- `wordFamily` 展示与主词具有真实派生关系的其他词。

两者功能不同，不要互相代替。

### synonyms 与 wordFamily

- `synonyms` 记录意义接近的词。
- `wordFamily` 记录派生关系。

同一个词不能因为“看起来相关”就随意放入两个字段。

### collocations 与 tip

搭配中已经自然体现的普通用法，不需要再在 `tip` 中重复解释，除非存在真正的易错点。

---

## 11. 增强版内容标准

用户说“增强版”“生成单词卡”“按单词卡格式整理”时，默认执行本规则。

增强版要求：

- 字段完整。
- 英式 IPA 正确。
- 每个词提供 1–5 个有教学价值的搭配。
- 例句与搭配严格对应。
- 有真实构词知识时补充 `morphology`。
- 有真实不规则形式时补充 `irregularForms`。
- 有真正近义词时补充 `synonyms`。
- 有真实、常用词族时补充 `wordFamily`。
- 有易混点时填写 `tip`。
- 不因字段存在就强行填充。
- 不降低内容密度到只满足最低可导入标准。
- 不用大量复杂内容让卡片超过学生理解范围。

---

## 12. 内容质量检查清单

生成或审核每张单词卡时，应检查：

### 基础字段

- [ ] `word` 拼写正确。
- [ ] 主词条使用合适的标准词形。
- [ ] `meaning` 与当前词性和音标一致。
- [ ] `pos` 使用统一缩写。
- [ ] `phonetic` 为英式 IPA。
- [ ] `emoji` 直观或为空字符串。

### morphology

- [ ] 当前词真实包含所写结构。
- [ ] 解释适合学生。
- [ ] 没有牵强拆词。
- [ ] 多个构词点已拆成多条。
- [ ] `knowledgeKey` 使用统一分类。

### collocations

- [ ] 数量为 1–5 个。
- [ ] 每个搭配自然、常见、有教学价值。
- [ ] 不同搭配没有重复同一用法。
- [ ] 每条例句真正使用对应搭配。
- [ ] 英文和中文语义一致。
- [ ] 例句难度适合学生。

### irregularForms

- [ ] 只写真正不规则的形式。
- [ ] 使用 `label` 和 `form`。

## 13. 禁止事项

禁止：

1. 使用旧字段 `en / zh / ex / note`。
2. 混用英式和美式 IPA。
3. 把普通相关词写进 `synonyms`。
4. 把拼写相似词写进 `wordFamily`。
5. 在 `irregularForms` 中填写规则变化。
6. 为凑数量生成低质量搭配。
7. 使用搭配与例句不对应的内容。
8. 用复杂、生僻表达增加表面信息量。
9. 在多个字段重复同一个知识点。
10. 强行拆解没有真实词缀的单词。
11. 因为字段为空就编造内容。
12. 把本文件中的 JSON 结构示例直接认定为最终导入格式。
