我要制作一个独立的“英语单词去重工具”。

这个工具不属于学生网站，不修改现有网站代码，也不修改现有导入器。它只用于我生成单词卡之前查询 Supabase，找出哪些单词已经存在。

## 目标

生成一个可以独立打开使用的本地 HTML 工具，例如：

```text
word-dedup.html
```

最好是完整独立 HTML，CSS 和 JavaScript 全部内联。双击文件后可以在浏览器中使用。

## 数据来源

工具需要读取我现有 Supabase 单词卡表中的 `word` 字段。

需要我后续提供或让 Codex 从现有项目中查找：

* Supabase URL
* Supabase anon key
* 单词卡表名
* `word` 字段名
* 当前项目使用 Supabase 的相关配置文件

工具只允许读取，不允许：

* 新增数据
* 修改数据
* 删除数据
* 覆盖已有单词

查询全部单词时必须处理分页，不能假设一次查询可以获得全部记录。

## 输入功能

页面提供一个大文本框，支持以下输入形式：

```text apple book chair
```

```text, apple, book, chair
```

```text/apple / book / chair
```

比较单词时统一：

* 去除前后空格
* 转成小写
* 清理多余标点
* 保留标准英文拼写用于显示

## 检查结果

点击“开始检查”后，告知：

1.需要人工确认的异常内容
2.  重复词
3. 新词

每组显示数量和具体内容。

以下情况放到“需要人工确认”，不要自动修正：

* 含数字
* 含异常字符
* 多个单词疑似粘连
* 输入了词组
* 拼写长度明显异常
* 无法可靠提取英文单词
* 类似 `gless`、`r d` 等疑似错误输入

## 页面功能

需要：

* Supabase 连接状态
* 已读取数据库单词数量
* 原始单词输入框
* 开始检查
* 清空
* 复制新词
* 显示本次输入、新词、重复词和异常项数量

## 页面风格

* 完整独立 HTML
* CSS、JavaScript 内联
* 简洁清楚
* 柔和配色
* 适合电脑/手机竖屏操作
* 结果区域容易复制
* 不需要儿童化动画
* 不依赖学生网站的 CSS 或 JavaScript

## 明确禁止

* 不修改学生网站
* 不修改现有导入器
* 不写入 Supabase
* 不增加登录系统
* 不生成单词卡内容
* 不生成音标、例句和中文释义
* 不设置数据库唯一约束
* 不主动重构其他项目代码
